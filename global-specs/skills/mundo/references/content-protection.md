# 内容保护铁律（血的教训）

**绝不删改蒙多项目的内容个性。** 布局只管排版和位置，不动文本内容。

## 规则

- 蒙多项目的口吻（"我是蒙多！蒙多想去哪就去哪！"）、等级制度（三省六部制）、中文优先结构 —— 是核心品牌资产
- 移动卡片位置 ≠ 重写内容
- 添加视觉效果（光晕、图标） ≠ 改文本
- 如需重写 README/子页面内容，必须先确认用户是否同意

## 教训（2026-05-25）

因"优化 README"把蒙多的等级制度、蒙多口吻、武器库表格全部删掉，替换成"SEO友好"的模板文案，导致用户强烈不满："最有特点的等级制度都被你删了，还原。"

## 正确流程

1. 修改前先备份：`git show HEAD:README.md > /tmp/backup-readme.md`
2. 只改布局/位置/样式，不动文本内容
3. 如需改内容，先问用户
4. 改完后 `git diff` 确认没有意外删除

## 主页卡片特殊效果

- 蒙多卡片使用金色光晕 `.card-mundo` + `@keyframes mundoGlow`
- 应用图标用真实图片（.icns 转 .png: `sips -s format png input.icns --out output.png`）
- 不用 emoji 替代真实图标
