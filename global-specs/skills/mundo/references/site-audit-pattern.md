# 全站审计模式

蒙多支持的并行审计工作流，适用于大规模项目页面审查。

## 触发条件
- 用户说"检查所有项目""审查全站""有哪些问题""优化所有页面"
- 新增多个项目页面后需要验证一致性

## 工作流

### 1. 扫描阶段
```
- 列出所有子目录（排除 .git / node_modules / __pycache__）
- 检查每个子目录是否有 index.html
- 标记"有页面"和"无页面"的项目
```

### 2. 并行审计
```
- 将页面列表拆分为 3-4 个组（每组 3-5 个页面）
- 用 delegate_task 并行派发审计子任务
- 每个子任务返回：问题清单（按高/中/低优先级分类）
- 合并所有子任务结果
```

### 3. 批量修复
```
- 高优先级：404 链接、缺失入口、安全风险 → 立即修
- 中优先级：内容不完整、教程不详细、路径错误 → 本会话修
- 低优先级：文案优化、样式微调 → 可顺带修
```

### 4. 收尾
```
- git add + git commit + git push
- 更新 skills/index.html（如有新增/删除 Skill）
- 同步 global-specs/（如有规范变更）
```

## 审计检查清单

每个页面需检查：
- [ ] 主页是否有入口链接？
- [ ] 下载链接是否 404？
- [ ] 教程步骤是否详细到点击位置？
- [ ] 语言是否通俗易懂（面向非技术用户）？
- [ ] 是否有 busuanzi 访问计数器？
- [ ] 免责声明到位了吗？
- [ ] 跨平台说明清晰吗（Win/Mac）？
- [ ] 外部 JS 依赖是否可用？

## 私有项目排除

用户标注为"非开源/家人专用"的项目不应出现在公开网址中：
- 从主页移除入口卡片
- 删除公开 index.html
- 代码可保留在仓库中，但页面不对公
