# 蒙多批量审计模式

当需要对整个项目进行全站审查时，蒙多可以同时分身多个子代理并行工作。

## 适用场景

- 网站全站链接/内容/教程审查
- 多文件代码质量检查
- 跨页面风格一致性修复

## 工作流

### 第一步：侦察（1 个代理）

用一个代理扫描所有目标文件，输出结构化问题清单（按严重程度分类：高/中/低）。

```
委托一个代理读取所有 index.html，返回问题清单表格
```

### 第二步：并行修复（3 个代理同时）

按问题类别分成 3 个并行任务：

1. **高优先级修复**：404 链接、缺失入口、下载失效
2. **内容增强**：页面太单薄的重写、添加教程/FAQ
3. **中等修复**：章节编号、命令错误、文本优化

每个代理接收明确的目标和具体文件路径，互不冲突。

### 第三步：提交

所有修改完成后 `git add -A && git commit && git push`。

## 关键原则

- 并行任务必须修改**不重叠的文件**，否则会产生冲突
- 每个代理给明确的文件路径和期望输出格式
- 先侦察后修复，不要边侦察边修
